大白话

正则是一种文本规则，使用这种规则可以快速地进行文本匹配，字符串处理。

正则表达式能干啥？

早期的正则

用于描述人类神经系统

支持计算机搜索算法

半个世纪之前世界上首款使用正则的qed编辑器问世

如今的正则

几乎得到了所有编程语言、编辑器和IDE的支持

爬虫、自动化脚本处理首选

它极大地提升了我的开发和工作效率

简直就是居家旅行、装逼耍酷必备神器

  • 元字符（反义元字符）
  • 限定符
  • 修饰符
  • 字符组（排除性字符组）
  • 多选结构
  • 括号
  • 转义字符

正则表达式语法

正则表达式家族

  • 基本的正则表达式( Basic Regular Expression 简称 BREs )
  • 扩展的正则表达式( Extended Regular Expression 简称 EREs )
  • Perl 的正则表达式( Perl Regular Expression 简称 PREs )

linux命令默认采取的正则分类

BREs：grep、sed

EREs：egrep、awk

  • 分组

  • 非捕获性分组
  • 命名分组

正则表达式在JS中的应用

ES6扩展了正则，新增如下两种修饰符：

  • y (粘连sticky修饰符)
  • u (支持4字节字符修饰符)

JS中共有7个方法与正则有关

  • compile
  • test
  • exec
  • match
  • search
  • replace
  • split

JS中使用正则表达式注意事项

难点：如何区分 exec 与 match方法？

鸡肋：search除了默认会将搜索子串转化为正则表达式形式，其它与indexOf方法无异

正则引擎

目前正则引擎有两种，DFA 和 NFA，NFA又可以分为传统型NFA和POSIX NFA。

JS正则引擎缺陷

  • 不支持固化分组、命名分组、占有量词优先
  • 不支持逆序环视
  • 4字节字符撇脚的支持性

如何快速的入门并熟练运用正则

  • 坚持使用正则处理文本

  • 及时查看文档，强化记忆

  • 技能点先点初阶再点高阶，循序渐进

  • 吃透分组是关键
